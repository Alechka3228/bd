# Общее задание (лабораторная работа 2‑7)

1. Загрузите эталонный проект и настройте его для ваших сервера и пользователя СУБД, а также установите нужные библиотеки для языка Python.
2. Замените базовые таблицы `people` и `phone` в проекте на указанные в вашем варианте, доработав все проверки целостности данных и соответствия типам данных.  
   - Не должно при добавлении и других действиях происходить ошибок PostgreSQL, приводящих к «вылету» программы или действиям без нормальных объяснений.  
   - Сообщения PostgreSQL о несоответствии типов данных, срабатывании ограничений целостности не являются нормальными объяснениями.  
   - Интерфейс ввода не должен использовать суррогатных ключей. Только понятные людям нетехнические данные или порядковый номер записи на экране.  
3. Доработайте для **первой таблицы** возможности удаления и редактирования сущностей (добавление, изменение, удаление с учётом связанных объектов второй таблицы).  
   - Интерфейс не должен использовать суррогатных ключей.  
4. Доработайте для **второй таблицы**:  
   - просмотр списка сущностей по указанному значению ключа из первой таблицы,  
   - добавление новых сущностей с этим ключом (включая все проверки целостности и типов),  
   - удаление сущностей в этом же интерфейсе.  
   - Интерфейс не должен использовать суррогатных ключей.  
5. Устраните все потенциальные SQL‑инъекции.  

**Внимание!** Нигде пользователь не должен вводить значения суррогатных ключей (например, `id`).  

Если сущности реализованы большим числом таблиц — разделите интерфейс на два шага: основные таблицы и связанные.  

Задания 6 и 7 — не обязательны, даны для саморазвития (постраничный просмотр, добавление UPDATE для второй таблицы).
